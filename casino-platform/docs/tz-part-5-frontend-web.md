## 2. Общие принципы Frontend

### 2.1. Технический стек

```
Next.js 14+          — App Router, SSR/SSG/ISR
TypeScript           — строгая типизация
Tailwind CSS         — стилизация
TanStack Query       — server state, кеширование запросов
Zustand              — client state (auth, wallet, UI)
React Hook Form      — формы
Zod                  — валидация форм
Axios                — HTTP клиент
Socket.IO client     — real-time (позже)
next-themes          — тёмная/светлая тема
```

### 2.2. Дизайн-концепция

Гибрид Stake и 1win:

- тёмная тема по умолчанию;
- акцентный цвет — выбирается на этапе дизайна (рекомендуется синий или зелёный);
- чистый layout без перегруза;
- яркие баннеры и промо-блоки в духе 1win;
- минималистичная навигация в духе Stake;
- мобильная версия — полноценная, не урезанная.

### 2.3. Цветовая схема


## 3. UI Component System

### 3.1. Базовые компоненты

Создать минимальную UI-библиотеку из компонентов, которые используются везде.

Каждый компонент — отдельный файл в `components/ui/`.

#### Button

**Варианты:**

- `primary` — заливка акцентным цветом;
- `secondary` — прозрачный с border;
- `ghost` — без border;
- `danger` — красный.

**Размеры:** `sm`, `md`, `lg`.

**Состояния:** `loading` (со спиннером), `disabled`.

#### Input

Варианты: `default`, `error`, `success`.

Дополнительно: `prefix` (иконка или текст слева), `suffix` (справа).

#### Select

Кастомный dropdown на базе Radix UI Primitives или headlessui.

#### Modal / Dialog

Базовый компонент модалки:

- backdrop blur;
- анимация появления;
- закрытие по Escape и клику вне.

#### Toast / Notification

Всплывающие уведомления:

- success, error, warning, info;
- позиция: правый нижний угол;
- auto-dismiss через 4 секунды.

#### Skeleton

Компонент-заглушка для состояния загрузки:

- прямоугольник, круг, текст;
- shimmer анимация.

#### Badge

Маленький тег: `new`, `hot`, `jackpot` и т.д.

#### Spinner

Анимированный индикатор загрузки.

#### Avatar

Круглый аватар с fallback на инициалы.

#### Tabs

Горизонтальные вкладки.

#### Pagination

Навигация по страницам.

#### Empty State

Компонент для пустых списков с иконкой и текстом.

#### Divider

Горизонтальная или вертикальная линия-разделитель.

#### Tooltip

Подсказка при hover.

## 5. Страницы аутентификации

### 5.1. Страница регистрации

**Маршрут:** `/register`

**Компонент:** `components/auth/RegisterForm.tsx`

**Поля формы:**

```
Email *
Пароль *                (показать/скрыть)
Подтверждение пароля *  (показать/скрыть)
Реферальный код         (optional, из URL если передан ?ref=CODE)
Чекбокс: Я соглашаюсь с условиями использования *
```

**Валидация (Zod):**

```
email         — валидный email
password      — минимум 8 символов, хотя бы 1 цифра
confirmPassword — совпадает с password
```

**UX:**

- показатель силы пароля (слабый / средний / сильный);
- при успешной регистрации — страница с сообщением «Проверьте email»;
- ссылка «Уже есть аккаунт? Войти»;
- разделитель «или»;
- кнопка «Войти через Google»;
- кнопка «Войти через Telegram».

**После отправки:**

- показать сообщение «Письмо с подтверждением отправлено на email»;
- не перенаправлять автоматически.

### 5.2. Страница входа

**Маршрут:** `/login`

**Поля формы:**

```
Email *
Пароль * (показать/скрыть)
Чекбокс: Запомнить меня
```

**UX:**

- кнопка «Забыли пароль?» → `/forgot-password`;
- кнопки Google и Telegram OAuth;
- при ошибке — показывать toast с сообщением;
- при успехе — редирект на главную или на страницу откуда пришли.

**Защита:**

- после 5 неудачных попыток — показывать captcha (опционально на MVP).

### 5.3. Подтверждение email

**Маршрут:** `/verify-email?token=...`

**Поведение:**

- при открытии страницы — автоматически отправлять запрос с токеном;
- если успешно — показать «Email подтверждён! Входим...» и редирект;
- если ошибка — показать причину и кнопку «Отправить письмо заново».

### 5.4. Забыли пароль

**Маршрут:** `/forgot-password`

**Поля:**

```
Email *
```

**UX:**

- после отправки — сообщение «Если аккаунт существует, письмо отправлено»;
- всегда показывать это сообщение независимо от результата (security).

### 5.5. Сброс пароля

**Маршрут:** `/reset-password?token=...`

**Поля:**

```
Новый пароль *
Подтверждение пароля *
```

**UX:**

- при успехе — редирект на `/login` с toast «Пароль изменён. Войдите»;
- при ошибке токена — «Ссылка недействительна или истекла» + ссылка на `/forgot-password`.

### 7.1. Фильтры

**Компонент:** `components/casino/GameFilters.tsx`

**Поведение:**

- фильтры сохраняются в URL params (`?category=slots&provider=pragmatic-play&sort=popular`);
- при изменении фильтра — URL обновляется, данные перезагружаются;
- кнопка «Сбросить фильтры»;
- счётчик найденных игр обновляется при фильтрации.

### 7.2. Бесконечная прокрутка

Использовать Intersection Observer:

- загружать следующую страницу при приближении к концу списка;
- показывать spinner при загрузке;
- при ошибке — кнопка «Попробовать снова»;
- показывать «Больше игр нет» в конце.

**Состояние 4: Запущена**

На мобильных:

## 22. Performance

### 22.1. Изображения

- использовать `next/image` для всех изображений;
- lazy loading для game thumbnails;
- placeholder blur пока грузится;
- WebP формат где возможно.

### 22.2. Code Splitting

- каждая страница — отдельный chunk;
- тяжёлые компоненты (игровой iframe, KYC загрузчик) — dynamic import;
- не включать в основной bundle библиотеки используемые только на одной странице.

### 22.3. Кеширование

- статичные страницы (главная) — ISR с revalidate 60 секунд;
- каталог игр — SSG или ISR;
- профиль, кошелёк — SSR или CSR с loading state.
